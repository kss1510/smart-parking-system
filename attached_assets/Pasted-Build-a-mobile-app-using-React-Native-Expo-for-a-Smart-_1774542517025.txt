Build a mobile app using React Native (Expo) for a Smart Campus Parking Management System using Supabase as backend.

Tech Stack:

* Frontend: React Native (Expo)
* Backend/Database: Supabase (PostgreSQL + Realtime + Auth)

Core Features:

1. Parking Zones (A, B, C)
2. Slots inside each zone
3. Real-time slot availability
4. Vehicle entry and exit system

Database Schema (Supabase):

* zones table:
  id, name

* slots table:
  id, zone_id, status (FREE, RESERVED, OCCUPIED), vehicle_number, entry_time, reserved_until

* history table:
  id, vehicle_number, slot_id, entry_time, exit_time

* users table (Supabase Auth)

Core Logic:

* When user selects a slot:
  → mark as RESERVED
  → set reserved_until = current_time + 30 seconds
* If not confirmed within 30 seconds:
  → automatically revert to FREE
* On confirmation:
  → mark slot as OCCUPIED
* On exit:
  → mark slot as FREE and store in history

Realtime:

* Use Supabase Realtime subscriptions to update slot status instantly across devices

App Screens:

1. Login Screen:

   * Simple login/signup using Supabase Auth

2. Dashboard Screen:

   * Show zones as cards
   * Each card shows:

     * Total slots
     * Available slots
     * Occupied slots
   * Color coding:

     * Green (available)
     * Red (full)
     * Yellow (limited)

3. Zone Detail Screen:

   * Grid view of slots
   * Each slot shows status:

     * FREE / RESERVED / OCCUPIED
   * Click slot to reserve or park

4. Parking Screen:

   * Enter vehicle number (or simulate QR input)
   * Show assigned slot
   * Button to confirm parking

5. Active Parking Screen:

   * Show current parking
   * Show parking timer (live duration)
   * Button to exit parking

6. History Screen:

   * Show last 5 parking records

Smart Features:

* Auto Assign Slot:
  → pick nearest available slot in least occupied zone
  → display "AI Suggested Slot"

* Notification Banner:
  → show when slot becomes available

* Admin Mode (toggle):

  * Add/remove slots
  * Force reset slot
  * Fix incorrect status

Offline Support:

* Cache slot data using AsyncStorage
* Show "Offline Mode" if no internet

UI Design:

* Clean modern mobile UI
* Use cards, icons, and color indicators
* Smooth navigation between screens
* Mobile-first UX

Extra:

* Simulate QR scan using input field
* Use timers for reservation countdown
* Show reserved countdown on slots

Constraints:

* Must run using Expo in Replit
* Include clear setup steps for Supabase keys
* Use environment variables for Supabase URL and API key
* Provide dummy data for demo

Goal:
Build a fully working mobile app prototype that demonstrates real-time parking management, smart slot allocation, reservation system, and user authentication for a hackathon demo.
